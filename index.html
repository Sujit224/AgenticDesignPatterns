The Five Patterns

Agentic Design Patterns

These patterns are not competitors. Real systems often combine them. A planning agent may use ReAct inside each step, and reflection may be added at the end for quality.

Pattern 01

Sequential Chain

Used when the steps are fixed and known in advance. Each step feeds directly into the next in a predetermined, reliable order. The LLM may power one or two steps, but the overall structure is deterministic.

Example: A resume parser — Upload → Extract skills → Categorize → Store → Summary. Same steps, every time.

✓ Use when

  • Process is predictable
  • Same steps every time
  • Little decision-making needed
  • Deterministic pipeline

✗ Avoid when

  • Next step depends on unknown results
  • Agent needs to reason about tools
  • Workflow changes per user input
STEP 1 → STEP 2 → STEP 3 → OUTPUT
📄 Upload Resume
🤖 Extract Skills (LLM)
🗂 Categorize Skills
💾 Store in Database
📊 Show Summary
Pattern 02

ReAct Pattern

Reason → Act → Observe → Repeat. The agent thinks about what to do, uses a tool, observes the result, then decides the next step dynamically. Perfect when the path isn't fully known upfront.

Example: Our e-commerce agent — it reasons, calls Order API, observes the status, reasons again, calls Return Policy, then answers.

✓ Use when

  • Exact path is not known
  • Agent needs external tools
  • Each step depends on prior results
  • Dynamic, adaptive tasks

✗ Avoid when

  • Workflow is already fixed
  • Deterministic pipeline works fine
  • No external tools needed
REASON → ACT → OBSERVE → REPEAT
❓ User Query
💭 Reason: "Need order status"
🔧 Act: Call Order API
👁 Observe: Out for delivery
💭 Reason: "Need return policy"
🔧 Act: Call Policy Tool
✅ Final Answer
Pattern 03

Planning Pattern

For big tasks where the agent should first create a roadmap, then execute. The agent decomposes the task, resolves dependencies upfront, and uses local reasoning per step.

Example: "Write a 5-page market research report on EVs." The agent first plans all 5 stages, then executes each one in order.

✓ Use when

  • Task has multiple stages
  • Structure can be identified upfront
  • Dependencies matter between steps
  • Complex, multi-part deliverables

✗ Avoid when

  • Task is very small/simple
  • Structure changes after every action
  • Plans become stale quickly
PLAN FIRST → EXECUTE STEPS
📝 "Write EV Market Research"
🗺 Create Plan (5 stages)
① Collect market size data
② Find top companies
③ Compare trends
④ Analyze challenges
📄 Final Report
Pattern 04

Reflection Pattern

The agent generates an output, critiques it, then refines it. A second pass checks facts, structure, and completeness. Use when quality matters more than speed.

Example: AI writes a project report. A critique pass checks accuracy, structure, and missing points. The answer is then improved before delivery.

✓ Use when

  • Quality matters more than speed
  • Clear evaluation criteria exist
  • Mistakes are costly (legal, medical)
  • Output needs fact-checking

✗ Avoid when

  • Task is simple or quick
  • Fast response is critical
  • No clear quality criteria
  • Latency/cost is a concern
GENERATE → CRITIQUE → REFINE
📝 Task: Write Report
✍ Generate Draft
🔍 Critique: Check facts?
🔍 Critique: Clear structure?
🔍 Critique: Missing points?
✏ Refine Draft
⭐ Polished Output
Pattern 05

Multi-Agent

Multiple specialized agents work in parallel under a coordinator. Use when the work is too large for one agent, needs different specializations, or benefits from parallel execution.

Example: Full software feature — Product, Coding, Testing, Security, and Review agents each handle their domain simultaneously.

✓ Use when

  • Task needs different specializations
  • Work is too large for one agent
  • Parallel work saves time
  • True scale is required

✗ Avoid when

  • One agent with tools can handle it
  • Coordination overhead > benefit
  • Debugging multi-agent is harder
  • Simple query / small scope
COORDINATOR → SPECIALISTS → MERGE
🏗 Build Software Feature
🎯 Coordinator Agent
📋 Product Agent
💻 Coding Agent
🧪 Testing Agent
🔒 Security Agent
🔀 Merge Results
🚀 Deployed Feature
